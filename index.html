Balises structurantes

Historiquement les textes étaient stockés dans des div.

On peut toujours en faire, mais l'arrivée de balise plus explicite que comme section ou article sont à privilégié.
Leurs comportement ne diffère pas du div mais elles rendent la lecture du fichier html plus simple.
Dans cette même logique, il existe des balises pour des zones de site : navbar, footer, aside, header,...

Une balise span existe aussi.
Elle fait comme la balise div à la différence qu'elle fonctionne en "inline" plutôt qu'en "block" (détail plus loin)

Un commentaire peut être glissé dans le fichier html.
Il ne sera pas exécuté par le navigateur, mais pourra être lu dans l'inspecteur
Pour ce faire, on encadrera ce qu'on veut commenter avec < !-- et terminera par -->

Le Cascade Style Sheet

On vu les balises html.
Avec le css, on peut leur changer leur apparance

Disposition

On peut placer du style dans nos balises html
C'est déconseillé (même si je viens de le faire pour mettre en rouge mon déconseillé).
La raison principale, c'est que ce style là est prioritaire comparé à celui qu'on retrouvera dans nos fichier css externe.

Dans notre fichier externe

Transition impecable pour le css dans un fichier externe!
On appelera ce fichier dans la balise head de note fichier html

Par une balise orpheline link, avec une propriété rel="stylesheet" et une indication de la source de fichier : href="./style/style.css"

Dans ce fichier, on indique les éléments qu'on veut modifier par un sélecteur, puis des accolades avec le style qu'on veut appliquer.
Chaque style devra être terminer par une ;
Et on peut commenter acomme en c# : avec // pour une ligne et /* pour plusieurs*/

Les sélecteurs

On peut indiquer un type de balise comme body.
On être plus précis en indiquant des classe en commençant par un point : .maClasse
#monId pour les identifiants.

On peut préciser notre sélecteur
Par exemple: body .maClasse pour toutes les balises avec la classe maClasse contenu dans mon body
Avec body > .maClasse, on limite aux enfants directement sous body
body + .maClasse touche aux premier enfant correspond
body img[alt] toujours au images avec un attribut alt

On peut mettre plusieurs sélecteurs si on les sépare par des ',' avant un block de style(entre les accolades).

Il y a un ordre de lecture des styles:

  • le style dans les balises html
  • le style sur les sélecteurs d'id du dernier fichier css indiqué
  • le style sur les sélecteurs d'id des premiers fichiers css indiqués
  • le style sur les sélecteurs de classe du dernier fichier css indiqué
  • le style sur les sélecteurs de classe des premiers fichiers css indiqués
  • ...

Formatage du texte

Font-size

Avec font-size, on peut gérer la taille du texte.
En fonction de "l'unité" choisit, la taille sera absolue (taille fixe) ou relative à la fenêtre ou la balise qui contient ce texte.

px pour les pixels va définir une taille fixe de notre texte.
%, em et rem va faire varier la taille en fonction des parents (rem se base sur la fonct-size de la balise html).

Gras

Avec font-weight : bold; on peut mettre en grase les éléments choisit

Italique

On peut mettre en italique avec font-style : italic;

Alignement horizontal

On jouer sur l'alignement horizontal avec text-align.
les valeurs possibles sont :

  • center
  • left
  • right
  • justify

Casse du texte

On peut aussi modifier la casse du texte grâce à text-transform
les valeurs possibles sont :

  • capitalize
  • uppercase
  • lowercase
  • none

mises en valeur du texte

text-decoration permet de jouer sur la mise en valeur du texte (comme les balises b, i , strong ...)
les valeurs possibles sont :

  • underline
  • line-through
  • overline
  • none

Les variables

Javascript gére ses variables manières dynamiques c'est à dire qu'il détermine lui-même le type de la variable.
En indiquant le nom de la variable et en lui assignant une variable avec un égal, il va "s'occuper le typage".
Exemple : maVariable = 5; Soucis cette variable à une portée globale
En commençant par let, on limite sa portée au block où il a été déclaré et ses enfants.
On verra aussi du var mais est déprécié

Du fait du typage dynamique, js rajoute une condition supplémentaire : === ou !==
En plus de vérifier l'égalité ou la différence, il vérifie si le type match entre les 2 éléments
Exemple: maVariable === 8;
Si maVarible = "8", on aura false en retour
